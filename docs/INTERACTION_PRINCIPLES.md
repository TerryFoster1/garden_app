# Interaction Principles

## Action Over Information

The user opens the app to decide or do something. Information supports action; it should not crowd the action.

## Fast Outdoor Loops

Common loops must be short:

- Notification -> Today -> Done.
- Today -> Scan -> Confirm -> Task.
- Scan -> Add Plant -> Bed -> Save.
- Alert -> Why -> Action.

## Scan First

Scan is a primary product identity. It should support:

- Identify plant.
- Diagnose issue.
- Identify pest.
- Identify weed.
- Track growth.
- Search by photo.

The scan flow should feel visual and immediate, not like data entry.

## Explainable Care

AI can recommend. Deterministic rules validate. The user confirms when confidence is uncertain or the action could affect plant health.

Task explanations should use practical language:

- "Skip watering because rainfall overnight was sufficient."
- "Cover basil tonight because the forecast is cold for tender transplants."

## Mobile Gestures

Future task cards should support swipe complete, snooze, and skip. Until gesture handling is added, large touch actions are preferred over tiny controls.

## Touch Everything Important

Important garden surfaces should feel interactive:

- Tap a bed to focus it.
- Tap a plant cluster to open plant detail.
- Tap a zone to shift context.
- Tap environmental state to understand why it matters.

Avoid static informational dead zones.

## Spatial Before Text

When a user can understand something visually, do not explain it first. Use text after the visual state establishes context.
